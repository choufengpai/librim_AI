## 1. 产品背景

### 1.1 现有产品概述

**librime** 是中州韵输入法引擎的核心库（版本 1.16.1），具有以下特点：

- **技术栈**: C++17、CMake、YAML配置
- **核心功能**: 拼写代数、词典系统、翻译器、过滤器
- **架构特点**: 模块化、跨平台、可扩展插件系统
- **支持方案**: 拼音、仓颉、五笔等多种输入方案
- **存储**: LevelDB、marisa-trie 字典树

### 1.2 目标平台与优先级

| 优先级 | 平台 | 前端应用 | 状态 |
|--------|------|----------|------|
| **P0** | Android | Trime (同文输入法) | 首期开发 |
| P1 | macOS | Squirrel (鼠须管) | 待扩展 |
| P2 | Windows | Weasel (小狼毫) | 待扩展 |
| P3 | Linux | ibus-rime / fcitx5-rime | 待扩展 |
| P4 | iOS | Hamster (仓) | 待扩展 |

**Android 前端选择**：
- **Trime**（已选定）- 主流 Android 前端，GitHub: https://github.com/osfans/trime
- 开发成本最低，与 librime 原生集成

**本地 LLM 选择**：
- **Qwen2.5-1.5B-Instruct**（已选定）- 阿里出品，中文能力强，体积小（~1.5GB）
- 推理引擎：MLC-LLM（移动端优化，有 Android SDK）

### 1.3 AI增强目标

基于 librime 输入法引擎，增加以下 AI 核心能力：
1. **个人特征识别** - 智能识别用户输入内容，提取个人特征
2. **智能输入增强** - 利用个人特征优化用户与 AI Bot 的交互
3. **隐私保护** - 提供无痕模式保护用户隐私

## 3. 详细功能规格

### 3.1 个人特征识别

#### 3.1.1 功能概述
- **功能描述**: 用户在日常打字时，输入法自动识别输入内容，提取对应的个人特征，并以非侵入式的方式询问用户是否要将该信息添加到个人信息库中
- **用户场景**: 
  - 用户在聊天中提到"我下周三有个会议"，输入法识别并提示添加日程
  - 用户提到"我不吃香菜"，输入法识别并提示添加饮食偏好
  - 用户提到"我家猫咪最近生病了"，输入法识别宠物信息
- **预期效果**: 
  - 非侵入式地构建用户画像
  - 用户完全可控，可选择是否保存
  - 长期积累形成精准的个人特征库

#### 3.1.2 个人特征分类

| 类别 | 子项 | 示例 |
|------|------|------|
| **基本情况** | 年龄、地区、社会身份、工作内容 | 28岁、北京、产品经理、互联网行业 |
| **兴趣偏好** | 食物偏好、娱乐偏好、旅行偏好、审美与风格 | 喜欢辣、爱看电影、偏爱海岛游、简约风格 |
| **生活习惯** | 作息时间、通勤方式、工作状态、运动与健身、家庭与宠物、社交活跃度 | 晚睡晚起、地铁通勤、远程办公、每周跑步、养猫、社交中等 |
| **近期事项** | 会议、旅行计划、预约 | 下周三会议、春节三亚旅行、牙医预约 |
| **社交关系** | 跟谁是什么关系 | 小明是大学同学、小红是同事 |

#### 3.1.3 技术方案
- **实现方式**: 本地 LLM 模型实时分析输入文本
- **涉及的模块**: 
  - 新增 `personal_profile` 模块
  - 本地模型推理引擎
  - 用户特征存储（加密本地数据库）
- **依赖项**: 
  - 本地轻量级 LLM（如 Gemma-2B、Qwen-1.8B 等）
  - LevelDB 或 SQLite 用于特征存储
- **API 变更**: 新增特征识别和存储相关 API

#### 3.1.4 Prompt 设计（初版）

```

你是一个用户个人特征识别助手。你的任务是从用户的输入文本中识别可能的个人特征信息。
```

#### 3.1.6 交互流程
```
1. 用户在任意应用中输入文字
2. 输入法检测到可能包含个人特征的文本
3. 本地模型分析文本，提取特征（置信度 > 阈值）
4. 在键盘工具栏显示轻量提示："识别到：[特征]，是否保存？"
5. 用户点击确认 → 保存到个人信息库
6. 用户点击忽略/滑走 → 丢弃该特征
7. 用户长期忽略 → 降低该类特征的提示频率
```

#### 3.1.7 备注
- 特征识别应在本地完成，保护用户隐私
- 提示频率需智能控制，避免打扰用户
- 支持用户手动编辑/删除已保存的特征

### 3.2 智能输入增强

#### 3.2.1 功能概述
- **功能描述**: 当用户在与 AI Bot 交互时，输入法自动根据个人特征库中的信息，智能补充用户输入，帮助用户获得更精准的回复或推荐
- **用户场景**: 
  - 用户在 AI 对话中输入"帮我规划三亚的过年旅行"
  - 输入法自动识别并展示可能相关的个人特征（如：喜欢辣食、全家出行、不会开车等）
  - 用户可删除不需要的特征，或手动添加其他特征
  - 确认后生成完整的增强 prompt 发送给 AI Bot
- **预期效果**: 
  - 减少用户重复描述个人偏好的次数
  - 提高 AI 回复的个性化和精准度
  - 用户完全可控，可选择使用哪些特征

#### 3.2.2 功能分期

| 阶段 | 功能 | 优先级 |
|------|------|--------|
| P0 | 自动特征匹配与展示 | 首期开发 |
| P0 | 用户删除不需要的特征 | 首期开发 |
| P0 | 生成增强后的输入内容 | 首期开发 |
| P1 | 手动选择/添加特征 | 后续迭代 |

#### 3.2.3 UI 设计规格

##### 特征建议卡片（P0）
- **位置**: 键盘上方，输入框下方
- **触发时机**: 用户在 AI 对话场景输入内容后
- **内容结构**:
  - 自动匹配的特征标签列表（横向排列）
  - 每个特征标签可点击删除（×按钮）
  - 加号按钮：手动添加更多特征（P1功能）
  - 发送按钮：确认并发送增强后的内容
- **交互**:
  - 点击特征标签的× → 移除该特征
  - 点击加号 → 拉起特征选择面板
  - 点击发送 → 生成增强 prompt 并发送

##### 手动选择特征面板（P1）
- **位置**: 底部弹出面板
- **内容结构**:
  - 顶部标题：「选择要添加的特征」
  - 分类展示：
    - 推荐特征（与当前输入弱相关）
    - 最近使用的特征
  - 每个特征可点击选中/取消
  - 确认按钮
- **交互**:
  - 点击特征 → 选中/取消选中
  - 点击确认 → 添加到当前输入的特征列表

##### 增强后的输入预览
- **展示方式**: 在发送前显示完整的增强内容
- **内容格式**: 原始输入 + 补充特征说明
- **示例**:
  ```
  用户原始输入：帮我规划三亚的过年旅行
  
  增强后内容：
  帮我规划三亚的过年旅行。
  补充信息：我喜欢吃辣的食物，会跟家里人一起前往，不会开车，会在公众假期开始前两天请假，偏好宽松的行程安排。
  ```

### 3.3 无痕模式

#### 3.3.1 功能概述
- **功能描述**: 提供一键开启的无痕模式，开启后输入法不会进行个人特征识别，保护用户隐私
- **优先级**: P1（后续迭代）
- **原型图**: 待补充
- **用户场景**: 
  - 用户需要输入敏感信息（密码、银行卡等）
  - 用户在私密对话中不想被记录
  - 用户暂时不想被识别特征
- **预期效果**: 
  - 一键快速切换
  - 无痕模式下完全不进行特征识别
  - 可设置自动关闭时间
- **开发说明**: 功能相对简单，可基于 PRD 规格直接开发，无需原型图

#### 3.3.2 技术方案
- **实现方式**: 
  - 键盘工具栏添加无痕模式开关
  - 开启后禁用特征识别模块
  - 可配置自动退出时间
- **涉及的模块**: 
  - `privacy_mode` 模块
  - 特征识别开关控制
- **依赖项**: 无
- **API 变更**: 新增无痕模式状态管理 API

#### 3.3.3 接口设计
```yaml
# 无痕模式配置
incognito:
  # 默认状态
  default_enabled: false
  
  # 自动关闭（分钟），0 表示不自动关闭
  auto_close_minutes: 30
  
  # 关闭时是否提示
  close_notification: true
  
  # 无痕模式下的图标样式
  icon_style: dark  # dark | light | accent
  
  # 视觉提示
  visual_hint:
    keyboard_tint: "#800000"  # 键盘微红色调提示
    border_indicator: true     # 边框指示器
```

#### 3.3.4 交互流程
```
1. 用户拉起键盘
2. 点击工具栏上的"无痕"图标
3. 键盘显示无痕模式已开启的视觉反馈（图标高亮/键盘色调变化）
4. 无痕模式下：
   - 不进行任何特征识别
   - 不记录任何用户输入到特征库
   - 正常输入功能不受影响
5. 用户再次点击图标 或 超过设定时间 → 退出无痕模式
6. 显示"无痕模式已关闭"提示
```

#### 3.3.5 备注

## 6. 测试要点（MVP）
